# UI Test Cases — Registration

**Формат:** ID · Название · Priority · Preconditions · Steps · Expected Result  
**Priority:** High / Medium / Low

---

## Позитивные сценарии

### TC-01 — Регистрация с валидными данными
**Priority:** High  
**Preconditions:** Открыта страница регистрации  
**Steps:**
1. Заполнить все обязательные поля валидными данными
2. Нажать «Зарегистрироваться»
**Expected Result:** Пользователь создан; отображается сообщение об успехе и/или редирект в личный кабинет

### TC-02 — Регистрация с паролем минимальной длины (8 символов)
**Priority:** High  
**Preconditions:** Открыта страница регистрации  
**Steps:**
1. Ввести пароль длиной ровно 8 символов
2. Заполнить остальные обязательные поля валидно
3. Отправить форму
**Expected Result:** Регистрация успешна

### TC-03 — Регистрация с email в верхнем регистре
**Priority:** Medium  
**Preconditions:** Открыта страница регистрации  
**Steps:**
1. Ввести валидный email в верхнем регистре
2. Ввести валидный пароль и остальные данные
3. Отправить форму
**Expected Result:** Регистрация успешна; email обрабатывается корректно (нормализация/сохранение по логике системы)

### TC-04 — Регистрация с пробелами в начале/конце email
**Priority:** Medium  
**Preconditions:** Открыта страница регистрации  
**Steps:**
1. Ввести `  user@mail.com  `
2. Заполнить остальные поля валидно
3. Отправить форму
**Expected Result:** Пробелы обрезаются или показывается корректная ошибка (в соответствии с логикой/требованиями)

### TC-05 — Двойной клик по кнопке «Зарегистрироваться»
**Priority:** High  
**Preconditions:** Открыта страница регистрации  
**Steps:**
1. Заполнить форму валидно
2. Дважды быстро нажать «Зарегистрироваться»
**Expected Result:** Создаётся один пользователь; нет дублей; UI не зависает

## Негативные проверки на безопасность ввода

**Preconditions:** Открыта страница регистрации

### TC-18 — SQL-инъекция паттерном в полях формы

**Steps:**

**Expected Result:** Нет ошибок сервера/утечек; корректная обработка ввода

## Уникальность и устойчивость

### TC-19 — Регистрация с уже существующим email
**Priority:** High  
**Preconditions:** В системе уже есть пользователь с данным email  
**Steps:**
1. Ввести существующий email
2. Заполнить остальные поля валидно
3. Отправить форму
**Expected Result:** Понятная ошибка (например, “Email уже используется”); новый пользователь не создаётся

### TC-20 — Обновление страницы во время отправки формы
**Priority:** Medium  
**Preconditions:** Открыта страница регистрации  
**Steps:**
1. Заполнить форму валидно
2. Нажать «Зарегистрироваться»
3. Быстро обновить страницу (F5)
**Expected Result:** Нет дублей/неконсистентного состояния; корректное поведение

### TC-21 — Нажатие Back после успешной регистрации
**Priority:** Low  
**Preconditions:** Регистрация успешно выполнена  
**Steps:**
1. Нажать “Back” в браузере
2. Попробовать повторно отправить форму (если возможно)
**Expected Result:** Нет повторного создания аккаунта; корректная обработка

### TC-22 — Отображение ошибок валидации рядом с полями
**Priority:** Medium  
**Preconditions:** Открыта форма регистрации  
**Steps:**
1. Ввести невалидные данные в несколько полей
2. Отправить форму
**Expected Result:** Ошибки показаны у соответствующих полей; тексты понятные; верстка не ломается
