# UI Test Cases — Access Control

**Формат:** ID · Название · Priority · Preconditions · Steps · Expected Result  
**Priority:** High / Medium / Low

---

### TC-31 — Доступ к защищённой странице без авторизации
**Priority:** High  
**Preconditions:** Пользователь не авторизован  
**Steps:**
1. Перейти по URL защищённой страницы (если известен) или открыть раздел, доступный только после логина
**Expected Result:** Редирект на страницу логина или отображение ошибки доступа; данные не раскрываются

### TC-32 — Открытие защищённой страницы после logout через “Back”
**Priority:** Medium  
**Preconditions:** Пользователь выполнил logout  
**Steps:**
1. Нажать “Back” в браузере
2. Попробовать открыть защищённый контент
**Expected Result:** Защищённый контент не отображается; при необходимости редирект на логин

### TC-33 — Истечение сессии (если реализовано)
**Priority:** Low  
**Preconditions:** Пользователь авторизован; есть таймаут сессии (если предусмотрено)  
**Steps:**
1. Оставить сессию неактивной до таймаута
2. Выполнить действие в приложении
**Expected Result:** Корректное завершение сессии/запрос логина; понятное уведомление
