# SQL Test Cases (Data Validation)

**Формат:** ID · Название · Priority · Preconditions · Steps · Expected Result  
**Priority:** High / Medium / Low

---

### TC-44 — SQL: пользователь создаётся в БД после регистрации
**Priority:** High  
**Preconditions:** Есть доступ к тестовой БД/дампу; известна таблица пользователей  
**Steps:**
1. Зарегистрировать пользователя через UI/API
2. Выполнить SQL-запрос поиска пользователя по email
**Expected Result:** Запись пользователя присутствует; ключевые поля заполнены корректно

### TC-45 — SQL: дубликаты по email не создаются
**Priority:** High  
**Preconditions:** Есть доступ к таблице пользователей  
**Steps:**
1. Попробовать зарегистрировать одного и того же пользователя дважды
2. Проверить количество записей по email
**Expected Result:** Ровно 1 запись на email

### TC-46 — SQL: пароль не хранится в открытом виде
**Priority:** Medium  
**Preconditions:** Есть доступ к данным пользователя в тестовой среде  
**Steps:**
1. Зарегистрировать пользователя
2. Проверить поле хранения пароля/хеша (если доступно)
**Expected Result:** Нет хранения plain text; хранится хеш/техническое значение согласно логике системы
