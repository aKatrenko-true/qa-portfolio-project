# API Test Cases (Postman)

**Формат:** ID · Название · Priority · Preconditions · Steps · Expected Result  
**Priority:** High / Medium / Low

---

### TC-37 — API: регистрация возвращает корректный HTTP-статус при успехе
**Priority:** High  
**Preconditions:** Доступен эндпоинт регистрации; подготовлено валидное тело запроса  
**Steps:**
1. Отправить запрос регистрации с валидными данными
**Expected Result:** Статус 200/201 (по логике API); ответ содержит корректные данные без чувствительных полей

### TC-38 — API: регистрация с невалидным email
**Priority:** High  
**Preconditions:** Доступен эндпоинт регистрации  
**Steps:**
1. Отправить запрос регистрации с невалидным email
**Expected Result:** 400/422 (по логике); в ответе понятная ошибка/описание

### TC-39 — API: регистрация с уже существующим email
**Priority:** High  
**Preconditions:** Пользователь с email уже существует  
**Steps:**
1. Отправить запрос регистрации с существующим email
**Expected Result:** 409/400 (по логике); ошибка “already exists”; новый пользователь не создаётся

### TC-40 — API: авторизация успешная
**Priority:** High  
**Preconditions:** Существующий пользователь; доступен эндпоинт логина  
**Steps:**
1. Отправить запрос логина с валидными данными
**Expected Result:** 200; возвращается токен/сессионные данные (по логике API)

### TC-41 — API: авторизация с неверным паролем
**Priority:** High  
**Preconditions:** Доступен эндпоинт логина  
**Steps:**
1. Отправить запрос логина с неверным паролем
**Expected Result:** 401/403 (по логике); корректная ошибка

### TC-42 — API: доступ к защищённому эндпоинту без токена
**Priority:** Medium  
**Preconditions:** Есть защищённый эндпоинт  
**Steps:**
1. Отправить запрос без токена/авторизации
**Expected Result:** 401; данные не возвращаются

### TC-43 — API: доступ к защищённому эндпоинту с валидным токеном
**Priority:** Medium  
**Preconditions:** Получен валидный токен  
**Steps:**
1. Отправить запрос с токеном
**Expected Result:** 200; данные возвращаются корректно
