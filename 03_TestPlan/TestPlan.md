# План тестирования

## 1. Общая информация

**Тип проекта:** Web application  
**Тип тестирования:** Ручное тестирование  
**Среда:** Test / Staging  
**Роль:** QA Engineer (Manual)

---

## 2. Цель тестирования
Целью тестирования является проверка корректности работы ключевого пользовательского функционала веб-приложения, выявление дефектов, влияющих на бизнес-логику, пользовательский опыт и стабильность системы, а также оценка готовности продукта к релизу.

---

## 3. Объект тестирования
Коммерческое веб-приложение с базовым функционалом интернет-магазина, включающим:
- каталог товаров
- карточку товара
- корзину
- оформление заказа
- пользовательскую авторизацию
- информационные модальные окна

---

## 4. Scope тестирования

### 4.1 Функциональное UI-тестирование
В рамках проекта тестируется следующий функционал:
- навигация по основным разделам приложения
- отображение каталога и фильтрация по категориям
- карточка товара (название, цена, описание, кнопка добавления в корзину)
- добавление и удаление товаров из корзины
- отображение итоговой суммы заказа
- оформление заказа через форму (модальное окно)
- пользовательская авторизация (регистрация, вход, выход)
- информационные модальные окна (контакты, информация о компании)

### 4.2 Негативное тестирование
- отправка форм с пустыми обязательными полями
- ввод невалидных данных (буквы, спецсимволы, пробелы)
- граничные значения полей
- повторные действия пользователя (многократные клики)

### 4.3 Mobile / Responsive тестирование (базово)
- проверка корректности отображения UI в мобильном разрешении
- доступность основных кнопок и элементов управления
- отсутствие блокирующих UI-дефектов

### 4.4 API тестирование (ограниченно)
- базовые проверки получения данных
- проверка структуры ответа
- проверка статус-кодов

> Примечание: API-тестирование выполняется в рамках pet-проекта и не покрывает все возможные сценарии.

---

## 5. Out of Scope
В рамках данного проекта **не выполняются**:
- нагрузочное тестирование
- глубокое security-тестирование
- автоматизированное тестирование
- тестирование интеграций со сторонними сервисами
- прямое тестирование базы данных (SQL используется как учебная практика)

---

## 6. Тестовое окружение
- Тип приложения: Web
- Браузер: Chrome (основной)
- Операционная система: Windows / macOS
- Среда: тестовая

---

## 7. Тестовые данные
- тестовые пользовательские аккаунты
- тестовые товары
- валидные и невалидные данные для форм
- данные для негативных сценариев

---

## 8. Риски
- нестабильность тестового окружения
- отсутствие требований к отдельным сценариям
- ограниченное покрытие нефункциональных аспектов
- различия поведения в разных браузерах

---

## 9. Критерии входа (Entry Criteria)
- приложение доступно
- основные страницы загружаются
- тестовое окружение стабильно

---

## 10. Критерии выхода (Exit Criteria)
- выполнены все тест-кейсы с приоритетом High
- критические дефекты задокументированы
- smoke-тестирование пройдено
- подготовлен итоговый отчёт о тестировании
