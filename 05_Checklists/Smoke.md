# Smoke Checklist — Web Application

## Цель
Быстрая проверка работоспособности критичного функционала приложения после сборки/деплоя.
Чек-лист используется для принятия решения о возможности дальнейшего тестирования.

---

## 1. Доступность приложения

- [ ] Приложение открывается по URL
- [ ] Страница загружается без критических ошибок
- [ ] Отсутствуют HTTP-ошибки 4xx / 5xx при открытии главной страницы
- [ ] Основные элементы интерфейса отображаются корректно

---

## 2. Регистрация пользователя

- [ ] Открывается страница регистрации
- [ ] Отображаются все обязательные поля формы
- [ ] Регистрация с валидными данными выполняется успешно
- [ ] После успешной регистрации пользователь получает подтверждение / редирект
- [ ] Регистрация с уже существующим email невозможна

---

## 3. Авторизация пользователя

- [ ] Открывается страница авторизации
- [ ] Авторизация с валидными данными успешна
- [ ] Авторизация с невалидными данными невозможна
- [ ] После авторизации пользователь попадает в личный кабинет

---

## 4. Пользовательская сессия

- [ ] Пользователь остаётся авторизованным при обновлении страницы
- [ ] Кнопка «Выход» отображается для авторизованного пользователя
- [ ] После выхода доступ к защищённым страницам закрыт

---

## 5. Основной пользовательский функционал

- [ ] Основной пользовательский функционал доступен после авторизации
- [ ] Данные пользователя корректно отображаются
- [ ] Ошибки пользовательского ввода обрабатываются корректно
- [ ] Нет критических UI-блокеров (невозможно нажать кнопку, форма не отправляется)

---

## 6. API (базовая проверка)

- [ ] API регистрации возвращает корректный HTTP-статус
- [ ] API авторизации возвращает корректный HTTP-статус
- [ ] В ответах API отсутствуют необработанные ошибки сервера

---

## 7. Мобильная версия (базово)

- [ ] Приложение открывается на мобильном устройстве / эмуляторе
- [ ] Форма регистрации отображается корректно
- [ ] Кнопки доступны для нажатия
- [ ] Критический функционал не ломается на мобильном разрешении

---

## Критерий прохождения Smoke-тестирования
- Все пункты с критичным пользовательским функционалом выполнены
- Отсутствуют блокирующие дефекты
- Приложение готово к дальнейшему функциональному тестированию
